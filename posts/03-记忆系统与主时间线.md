## 一、角色记忆系统（identifier 11）

### 1.1 核心理念

> **记忆不是客观记录，是角色的主观体验。**

传统的 AI 对话总结是上帝视角的白描压缩——"A 做了什么，B 说了什么"。角色记忆系统的根本区别在于：**每个角色用自己的眼睛看世界，用自己的方式记住事情。** 同一件事，不同角色记下的内容完全不同。

三条核心法则：

| 法则 | 含义 |
|------|------|
| **记忆是主观的** | 角色的性格调色盘决定了她怎么理解、怎么记忆一件事 |
| **记忆是非全知的** | 角色只能记住自己看到的、听到的、感受到的 |
| **记忆是有情感的** | 人记住的不是事件的完整信息，而是那个瞬间最强烈的感受 |

**同一顿饭的三个角色记忆**：

```
喜欢{{user}}的角色：
  "他给我倒了水，手指碰到杯子的时候犹豫了一下。"

讨厌{{user}}的角色：
  "又在装好人。倒水的时候特意看了我一眼，恶心。"

无所谓的角色：
  （可能根本不记这顿饭）
```

---

### 1.2 五种记忆形式

记忆形式由角色的性格和习惯决定——冷酷的人不会写日记体，活泼的人不会用片段闪回。同一个角色可以混用多种形式。

#### 形式一：物品关联型

以具体物品为锚点展开。适合**重视实体、有收集癖、对物品有情感寄托**的角色。

```
银色发卡：是他在夏祭上从摊位买的。他比了半天不知道选哪个，最后我
说了句"那个银色的好看"，他就立刻买了。别在头发上的时候他的手在抖。
我现在每天都戴着。
```

#### 形式二：日记型

以日期和内心独白的形式。适合**内向、有写日记习惯、内心戏丰富**的角色。

```
三月初的某天：今天和他一起去了旧书店。他翻书的时候特别安静，我偷
偷看了他好几次都没被发现。后来他突然抬头说"你在看什么"，我说"在
看书"，其实我在看他。笨蛋。
```

#### 形式三：片段闪回型

以感官碎片的形式。适合**经历过创伤、记忆偏直觉型**的角色。

```
咖啡的味道：那天的咖啡是苦的。他坐在对面，一直搅那杯拿铁但没喝。
我说了什么让他不高兴了。好像是关于他父亲的事。他最后说"没事"，
但搅拌勺停了。
```

#### 形式四：关系定位型

以对另一个人的评价和关系认知为核心。适合**社交敏感、关系导向、警惕性强**的角色。

```
关于角色B：一开始觉得她很碍眼，总是凑在他身边。后来一起被困在暴
雨里，她把伞让给了我而不是他——那时候我才意识到她不是在跟我抢，
她只是也很喜欢他而已。现在我不讨厌她了，但还是不想让她离他太近。
```

#### 形式五：碎念型

口语化的内心独白。适合**大大咧咧、想法直白、不善整理思绪**的角色。

```
上周还是上上周来着，和他打了一场篮球，他竟然赢了我一球！一球！明
明之前从来没赢过！我觉得他偷练了但他死不承认。下次一定赢回来。
```

**形式选择与调色盘的联动**：

| 调色盘底色 | 倾向的记忆形式 |
|-----------|--------------|
| 叛逆 | 碎念型（强调不服从、对抗、独立） |
| 依赖 | 物品关联型/日记型（强调与依赖对象的互动细节） |
| 坏女人 | 关系定位型（戏谑、居高临下的口吻） |
| 温柔 | 日记型（关注他人状态和感受） |
| 执念 | 片段闪回型（反复回忆关键场景） |
| 精明 | 关系定位型（分析性的人际评估） |

---

### 1.3 记忆的筛选规则

#### 值得记住的

1. **有情感关联的事件** — 开心、难过、愤怒、害怕、心动
2. **改变认知的事件** — 发现某人和自己想的不一样、了解新信息
3. **主动参与的重要事件** — 战斗、危机、重要选择

#### 不记或简略记

1. 角色**不在场**的事件 → 完全不记
2. 与角色**无关**的背景事件 → 不记或一句话带过
3. **重复发生**的日常 → 压缩为习惯性记忆（"他每天早上都会给我倒水"）
4. 角色**不在意**的事 → 不记（性格决定什么是"不在意"）

---

### 1.4 记忆衰退机制

#### 三层衰退梯度

| 时期 | 特征 | 示例 |
|------|------|------|
| **近期**（最近几轮） | 细节清晰，情感鲜活 | "昨天他在图书馆帮我找了一本书。蓝色封面。他从最上面的架子够下来的，差点摔了。他把书递给我说'这本吧？'，我说'嗯'，但其实我想说谢谢。" |
| **中期**（一段时间后） | 细节模糊，情感沉淀为判断 | "之前他在图书馆帮我找过一本书。好像是什么蓝色封面的。他很认真地找了好久。" |
| **远期**（很久之后） | 只留核心，结论化 | "他会帮我做一些小事。" |

#### 阻止衰退的三种锚点

1. **极度重要**的记忆不衰退 → 告白、背叛、生死瞬间
2. **反复回想**的记忆不衰退 → 角色经常想起的事
3. **有实物锚定**的记忆衰退更慢 → 收到的礼物、留下的伤疤

---

### 1.5 记忆篡改机制

这是整个记忆系统中最具洞察力的设计。

#### 关系改善后的记忆修正

```
之前讨厌角色B时的记忆：
  "角色B今天又凑过来了，故意在他面前表现。真虚伪。"

和角色B关系改善后，同一段记忆变成：
  "之前角色B经常在他身边。那时候我不理解她，现在想想她只是也喜欢他。"
```

注意：不是删除原来的记忆，而是**同一件事的解读变了**。

#### 关系恶化后的记忆扭曲

```
之前关系好时的记忆：
  "他送了我一条围巾。很暖和。"

关系恶化后：
  "他以前送过我东西。现在想想，可能只是在收买人心。"
```

#### 创伤后的记忆封锁/变形

太痛苦的记忆可能被角色无意识地：
- **封锁**："那天发生了什么我不太记得了。"
- **美化**："其实也没有那么严重。"
- **替换**："应该是我自己不小心的。"

---

### 1.6 跨角色非全知原则

**场景**：{{user}}、角色A、角色B三人一起吃饭。饭后{{user}}送角色A回家。

```
角色A的记忆：
  "今天和他还有角色B一起吃了饭。角色B一直在说话，有点吵。不过饭
  后他送我回家了，路上只有我们两个人。他说了句'今天开心吗'，我说
  '嗯'。其实特别开心。"

角色B的记忆：
  "今天吃饭的时候聊了很多。角色A话很少，不知道在想什么。吃完饭我
  先走了，好像他送角色A回去了——嗯，果然。"
```

核心约束：
- 角色A不知道角色B先走后的想法
- 角色B不知道{{user}}送角色A回家路上说了什么
- 两人对同一顿饭的侧重点完全不同

---

### 1.7 记忆区格式与容量

```
[角色名]的记忆:

[近期记忆]
（最多 8 条，每条 50-150 字，按时间倒序）

[沉淀记忆]
（最多 8 条，每条 20-60 字）

[核心记忆]
（最多 5 条，每条 20-80 字，永不丢弃）
```

**总容量**：单个角色 ~500-1500 字

**衰退流向**：近期满 8 条 → 最旧压缩移入沉淀 → 沉淀满 8 条 → 不重要的丢弃，重要的浓缩到核心

## 二、主时间线系统（identifier 13）

### 2.1 核心定位

> **时间线是故事的骨架，角色记忆和人设是血肉。**

时间线与角色记忆形成严格互补——**时间线只记客观事实，不记主观感受**。

| 时间线记录的 | 不记录的（归其他系统） |
|------------|-------------------|
| 谁做了什么（行为事实） | 角色怎么想的 → 记忆系统 |
| 在哪里发生的（空间事实） | 角色的感受 → 记忆系统 |
| 什么时候发生的（时间事实） | 事件的意义 → 人设更新 |
| 导致了什么结果（因果事实） | 对话详细内容（除非关键） |

**对比示例**：

```
时间线记录：
  "{{user}}在图书馆帮角色A从最高层书架取下一本蓝色封面的星座书。"

角色A的记忆：
  "他找了很久，从最上面的架子够下来的，差点摔了。他把书递给我的时
  候说'这本吧？'，我说'嗯'，但其实我想说谢谢。"
```

时间线是骨头，记忆是血肉。

---

### 2.2 事件 ABC 三级分类

| 级别 | 定义 | 判断标准 | 处理方式 |
|------|------|---------|---------|
| **A 级（转折点）** | 改变故事走向的关键事件 | 关系质变 / 世界不可逆改变 / 核心处境改变 / 重大选择 | **永不压缩** |
| **B 级（推进点）** | 推动剧情但不改变根本走向 | 获得新信息 / 完成阶段任务 / 建立新联系 / 有意义的移动 | 远期压缩为一句话 |
| **C 级（填充）** | 日常互动，没有推进意义 | 日常对话 / 重复行为 / 无后续影响 | **不记录** |

C 级的例外：同类事件反复出现时合并为习惯记录（"这段时间{{user}}和角色A每天一起吃午饭"）。

---

### 2.3 时间线的三区域结构

#### 区域一：当前态势

每次更新都**完整重写**，始终反映最新状态：

```yaml
当前态势:
  时间: [故事中的当前时间]
  地点: [当前场景]
  在场角色: [目前在场的角色列表]
  当前状况: [一两句话描述正在发生什么]
  核心矛盾: [当前推动剧情的主要冲突或目标]
```

容量：50-100 字

#### 区域二：活跃线索

正在发展中、尚未结束的事件线：

```yaml
活跃线索:
  - 线索名: [简短命名]
    起因: [怎么开始的]
    进展: [现在发展到哪了]
    涉及角色: [相关角色]
    悬念: [还没揭晓的问题]
```

管理规则：
- 新悬念 → 新增线索
- 线索解决 → 移入已发生事件，从活跃线索移除
- 长期无进展 → 标注 [休眠] 但不删除
- 最多 **8 条**，超出时合并或休眠

每条 50-100 字

#### 区域三：已发生事件

按时间倒序排列：

**A 级事件格式**：
```
[时间] [A级] 事件标题
  发生: （具体描述：谁在哪做了什么，结果是什么）
  影响: （改变了什么——世界状态、关系格局、后续走向）
  关联线索: （触发/解决/改变了哪条线索）
```

**B 级事件格式**：
```
[时间] [B级] 事件摘要
  （一两句话描述事件内容和结果）
```

---

### 2.4 时间线压缩规则

| 时期 | A 级处理 | B 级处理 |
|------|---------|---------|
| **近期** | 完整保留 | 完整保留 |
| **中期** | 完整保留（永不压缩） | 压缩为一句话摘要 |
| **远期** | 保留但可简化影响描述 | 多条合并为时间段摘要 |

**压缩示例**：

```
近期（完整）：
  [第三天上午] [B级] {{user}}在集市发现出售灵草的摊位
    角色A识别出其中一株是罕见的九叶还魂草，建议购买。
    {{user}}花了三百灵石买下。

中期（一句话）：
  [第三天] {{user}}在集市购买了九叶还魂草（角色A建议）

远期（合并）：
  [第三天至第五天] {{user}}在城内补给，购入九叶还魂草等物资
```

**容量上限**：
- 近期事件：8-12 条完整记录
- 中期事件：15 条压缩摘要
- 远期事件：10 条合并摘要
- **总容量：~1500-3000 字**

## 三、三系统联动

时间线、角色记忆、角色人设更新三个系统各有分工：

```
                    ┌─────────────────────┐
                    │    主时间线系统      │
                    │  "发生了什么"        │
                    │  客观事实 / ABC分级  │
                    └──────┬───────┬──────┘
                           │       │
              A级事件触发  │       │  提供事实锚点
                           ▼       ▼
              ┌────────────────┐  ┌───────────────────┐
              │  角色人设更新   │  │  角色记忆系统      │
              │ "因此变成什么人"│  │ "怎么理解这件事"   │
              │  调色盘/三面性  │  │  主观/有情感/非全知 │
              └────────────────┘  └───────────────────┘
```

**分工原则**：
- "发生了什么" → 时间线
- "怎么理解" → 记忆
- "因此变成了什么人" → 人设

**实际执行**：EW 的三个 Flow 并行运行（dispatch_mode: parallel），各自独立向 AI 发送请求，各自获得 JSON 输出，各自写入对应的世界书条目。Flow 之间不互相依赖。

## 四、Flow 2 记忆更新思维链（identifier 12）

七步结构化思考：

### 步骤一：读取角色人设
- 每个在场角色的调色盘是什么？底色、主色调、点缀？
- 角色与 {{user}} 的关系？与其他角色的关系？

### 步骤二：扫描最新楼层
- 发生了什么？谁在场？谁不在场？
- **不在场的角色这轮不产生记忆**

### 步骤三：以每个在场角色的视角判断
- 这件事对这个角色重要吗？
- 重要 → 角色怎么理解？善意/恶意/中性？（由关系+调色盘决定）
- 角色会注意什么细节？（由底色和主色调决定关注焦点）
- 用什么形式记住？（五种形式中选择）

### 步骤四：记忆衰退检查
- 近期满 8 条？→ 最旧的压缩移入沉淀
- 沉淀满 8 条？→ 不重要的丢弃，重要的浓缩到核心
- 可合并的重复记忆？

### 步骤五：记忆篡改检查
- 角色与某人的关系最近变化？→ 旧记忆需要重新解读？
- 经历创伤？→ 记忆封锁/美化/替换？

### 步骤六：人格一致性自检
- 记忆是否符合角色调色盘风格？
- 是否用了全知视角？（记录了角色看不到的事）
- 是否所有角色记忆语气都一样？（不应该）
- 是否有角色不可能注意到的细节？

### 步骤七：输出
- 每个需要更新记忆的角色及其新增/修改条目
- 不需要更新的不输出

## 五、Flow 3 时间线更新思维链（identifier 14）

七步结构化思考：

### 步骤一：扫描
列出所有可观察到的事件、时间、地点、参与角色

### 步骤二：事件分级
- A 级？关系质变 / 世界不可逆改变 / 核心处境改变 / 重大选择
- B 级？新信息 / 阶段任务 / 新联系 / 有意义的移动

### 步骤三：线索追踪
- 新悬念 → 新增线索
- 推进已有线索 → 更新进展

- 近期 >12 条 → 最旧B级压缩移入中期
- 中期 >15 条 → 合并为时间段摘要移入远期

- 是否混入主观感受？（"他很伤心" → 不应出现）
- 是否有评价性语言？（"这是重要的一步" → 不应出现）
- 是否有文学化修辞？（时间线是档案，不是小说）

## 六、禁止事项汇总

### 记忆系统禁止

1. 禁止以全知视角记录（角色不在场的事不记）
2. 禁止白描零度风格（记忆**必须有**角色的人格印记）
3. 禁止流水账（"先做了A，然后做了B"）
4. 禁止所有角色用同一种语气记忆
5. 禁止记录角色不在意的琐事

### 时间线系统禁止

1. 禁止记录角色的主观感受和内心想法
2. 禁止对事件进行评价或解读
3. 禁止记录私密对话细节（除非关键行为）
4. 禁止重复记录同类C级事件
5. 禁止使用文学化修辞

**注意两个系统的禁止事项恰好互补**：记忆禁止白描零度（必须有人格），时间线禁止主观感受（必须客观）。这是设计上的有意为之——让两个系统各守一边，互不越界。

## 七、identifier 16「动态世界记录思维链」——废弃的全合一方案

identifier 16 在所有三个 Flow 中均为 OFF，但它的内容揭示了作者最初的设计思路——一个统一的五阶段思维链：

```
阶段一：场景扫描（共用前置分析）
阶段二：角色动态更新
阶段三：角色记忆更新
阶段四：主时间线更新
阶段五：controller_model配置
阶段六：综合输出（JSON）
```

这个全合一方案被拆分为三个独立 Flow（角色更新/记忆/时间线）的原因可能包括：
1. **token 开销**：一次性执行三个子系统对上下文的消耗太大
2. **失败隔离**：一个子系统出错不影响其他两个
3. **并行加速**：三个 Flow 可以并行请求，缩短总耗时
4. **调试便利**：可以单独开关某个子系统
