## 二、好感度体系

世界书中有一条蓝灯常驻条目 `角色好感度阶段参考`（constant=true, enabled=true），定义了完整的双轨好感度系统。

### 2.1 基本参数

- **范围**：[-1000, 1000]
- **铁律**：男性角色只有兄弟情义，**绝对禁止**男同爱情/暧昧/道侣

### 2.2 男性好感度体系（兄弟情义线）

| 阶段 | 范围 | 关系本质 |
|------|------|---------|
| 仇敌/生死大仇 | [-1000, -500] | 不死不休 |
| 厌恶/敌意 | [-500, -200] | 强烈反感 |
| 陌生/冷淡 | [-200, 0] | 戒备，公事公办 |
| 素不相识 | [0, 100] | 礼貌客套 |
| 萍水相逢 | [100, 200] | 略有好感 |
| 点头之交 | [200, 300] | 打招呼聊江湖事 |
| 泛泛之交 | [300, 400] | 喝酒聊修炼心得 |
| 相识 | [400, 500] | 信任初建，并肩作战 |
| 相知 | [500, 600] | 可以交托后背 |
| **知己** | [600, 800] | 肝胆相照。一壶酒、一场战、一声"兄弟" |
| **莫逆之交** | [800, 900] | 生死相托。一个眼神便知心意 |
| **生死之交** | [900, 1000] | **终点**。可为对方舍命，千年不见依然如故 |

### 2.3 女性好感度体系（友情→爱情线）

| 阶段 | 范围 | 关键设计 |
|------|------|---------|
| 仇敌→冷淡 | [-1000, 0] | 同男性 |
| 素不相识→相知 | [0, 600] | 同男性 |
| **知己** | [600, 700] | 纯粹的友情巅峰，**不含爱意** |
| **红颜知己** | [700, 800] | 超越友谊但不一定是爱情。**本身就是美好的终点** |
| **情愫暗生** | [800, 900] | **必须有特定契机**才能从红颜知己突破 |
| 两情相悦 | [900, 950] | 互相倾心 |
| 道侣 | [950, 980] | 需正式仪式或生死誓言 |
| 双修道侣 | [980, 990] | 灵魂交融 |
| 生死相依 | [990, 995] | 需经历生死劫难 |
| 不离不弃 | [995, 1000] | 修仙界最高层次，极其罕见 |

### 2.4 关键设计点

**从红颜知己到情愫暗生的四种契机**：
1. 生死与共的经历
2. 长时间朝夕相处（数月甚至数年）
3. 某个特定瞬间的心动
4. 被其他异性接近时产生的嫉妒

> "红颜知己≠爱情，这是正统修仙世界的重要特征。冷淡角色即使好感度800，也可能只是'认可你，视你为知己'。温柔角色好感度750可能已心生情愫。"

这条规则确保了**AI 不会让所有女角色最终都爱上用户**——红颜知己停住是完全正常的。

**修仙世界特殊原则**：
- 感情发展以**年**计（修士寿元漫长）
- 道心优先（许多修士将修道置于儿女情长之上）
- 红颜知己比道侣更常见

## 三、极阳天尊暗线系统（44 条条目）

### 3.1 暗线总架构

暗线是整个修仙世界中隐藏最深的叙事层——讲述远古极阳天尊牺牲自身创世的故事。玩家不会被告知有暗线存在，只会在游历世界时自然地听到碎片，最终在红尘酒家（起点也是终点）拼出完整的真相。

```
                    暗线_总规则
                        │
        ┌───────────────┼───────────────┐
        │               │               │
   第一层：起点涟漪  第二层：传说涟漪  第三层：遗迹涟漪
   （红尘酒家出发）  （世界各处碎片）  （物理痕迹发现）
        │               │               │
        └───────────────┼───────────────┘
                        │
                   第四层：真相涟漪
                   （核心真相持有者）
                        │
                   第五层：终点涟漪
                   （回到红尘酒家）
```

### 3.2 五层涟漪详解

#### 第一层：起点涟漪（暗线_主线01_起点涟漪）

- **地点**：红尘酒家
- **核心角色**：洛红尘（极阴天尊的凡人化身，知道全部真相但绝不主动说破）
- **机制**：{{user}} 来红尘酒家讲故事/喝酒时，洛红尘会用"你的故事让我想起一个人"的方式随意提到远古往事，并推荐"有趣的地方"或"有趣的人"

洛红尘的 7 条推荐示例（根据情境选择）：
1. "剑鸣城外十万大山里有棵老槐树" → 引导去**老树精**
2. "天启城铁匠铺的赵铁柱" → 引导去**赵铁柱**
3. "玄天宗有个小姑娘特别会算卦" → 引导去**韩青衣**
4. "流金城万宝商会金库最深处" → 引导去**萧远山**
5. "西漠灵山脚下有个茶铺" → 引导去**莲华**
6. "北冥海深处最后一头鲲" → 引导去**鲲**
7. "东荒涂山的小九" → 引导去**涂山小九**

**洛红尘绝不会做的事**：
- 说出"极阳天尊"三个字
- 解释为什么推荐这些地方
- 表现出焦急、期待、催促
- 暗示 {{user}} 有"使命"或"任务"

#### 第二层：传说涟漪（暗线_主线02_传说涟漪）

覆盖几乎所有角色和地点——碎片类型有 7 种：

| 碎片类型 | 来源角色 | 示例 |
|---------|---------|------|
| 口传传说 | 年长角色、凡人、说书人 | "我听老一辈说过……" |
| 宗门秘典 | 宗门长老、掌门 | "我们宗门古籍里记载……" |
| 种族记忆 | 妖族、龙族长老 | "我们族群的血脉记忆中……" |
| 亲眼所见 | 真仙境老怪物 | "我活了几万年，亲眼见过……" |
| 遗迹发现 | 探险者、散修 | "我在某个遗迹里发现了……" |
| 异象感应 | 特殊体质者、天机阁 | "我能感觉到某种力量……" |
| 修炼感悟 | 各道高阶修士 | "我修炼到极致时感受到……" |

**碎片密度分布**：
- 剑鸣城及周边：最多最容易获取（凡人传说）
- 各大宗门：中等密度（需建立关系）
- 妖族领地：种族记忆（信任后分享）
- 魔道势力：冲突后获得（战斗感慨/嘲讽/遗言）
- 地府：最核心最完整（但难以到达）
- 散修/特殊角色：随机散落

#### 第三层：遗迹涟漪（暗线_主线03_遗迹涟漪）

不是新区域，而是散布在已有区域中的物理痕迹——石碑、金色光芒、温热法器、地底脉动。4 个遗迹条目：

| 遗迹 | 触发关键词 | 说明 |
|------|-----------|------|
| 阳极石碑 | 石碑 | 古老石碑上的符文 |
| 金光裂隙 | 金光、裂隙 | 空间中的金色光芒 |
| 轮回井 | 轮回、枯井 | 地府中的古老井 |
| 阳纹法器 | 阳纹、金色花纹 | 带有神秘金色纹路的法器 |

#### 第四层：真相涟漪（暗线_主线04_真相涟漪）

仅在面对**核心真相持有者**时加载——秦无伤、清奶、焚天、十殿阎王等太古存在，以及进入阴曹地府时。

#### 第五层：终点涟漪（暗线_主线05_终点涟漪 + 暗线_终章_洛红尘归来）

回到红尘酒家，与洛红尘的安静重逢。起点即终点。

### 3.4 暗线碎片实例

以**灵霄宗青云宗噬心殿**碎片条目为例，展示 5 个角色如何从不同角度指向同一核心：

| 角色 | 碎片来源 | 碎片内容核心 | 指向 |
|------|---------|------------|------|
| 云倾颜（太阴灵血） | 特殊体质感知 | "太阳里有一个人。不是真的在太阳里，是他变成了太阳的一部分" | 极阳天尊化为天地 |
| 风烬雪（失明剑修） | 灵魂视觉 | "每次濒死时，那道金色的光就会出现。不会救我，但让我不害怕" | 极阳天尊守护每个灵魂 |
| 玉横枝（噬心殿主） | 毒理学研究 | "死亡是被人设计出来的。那个循环是人造的，精密得令人发指" | 极阳天尊创造轮回的代价 |
| 苏妍（灵霄宗弟子） | 日常感应 | "每次看太阳的时候，我都觉得太阳在看我" | 极阳天尊的残留意识 |
| 楼玄烬（噬心殿） | 吞噬记忆 | "每个人的灵魂深处都藏着同一道金光。那道光不是记忆，是印记" | 极阳天尊给灵魂的保护 |

**AI 演绎规则**：
1. 碎片出现必须自然——闲聊/回忆/感慨时带出
2. 角色的碎片必须符合其身份和认知层次
3. 每个碎片自然指向其他角色或地点，形成引导链
4. 所有碎片角色都**不知道完整真相**——他们只知道自己那一小块

## 附录：暗线、好感度和动态事件在运行时到底怎么工作？

### D1. 暗线系统的完整运行时链路

暗线从**不被激活**到**碎片出现在玩家面前**，经历了以下精确步骤。以"玩家在玄天宗闲逛时听到韩青衣随口提到远古传说"为例：

```
玩家发送："我在玄天宗四处转转，看看有什么有趣的人"
  │
  ▼ 动态内容控制器执行
  
Step 1: 变量读取
  currentDomain = '中央神州'
  currentArea = '星辰山脉'
  currentScene = '玄天宗'
  currentEvent = ''  (没有事件)
  
Step 2: 角色检测
  消息中未提到具体角色名 → detectedCharacters = []
  （但韩青衣是玄天宗的，AI 可能在回复中提到她）

Step 3: 事件检测
  currentEvent = '' → 所有 14 种事件都不匹配
  → eventGuideLoaded = false ← 关键！

Step 4: 暗线触发检测
  if (!eventGuideLoaded) {  ← 事件未加载，进入暗线检测
    
    // 检测暗线核心角色
    darklineCharacters = ['洛红尘','秦无伤','清奶','焚天',...]
    → detectedCharacters 为空 → 不触发
    
    // 检测暗线地点
    textToSearch = '星辰山脉|玄天宗|中央神州'
    darklineLocationKeywords 包含 '玄天宗'
    → '星辰山脉|玄天宗|中央神州'.includes('玄天宗') = true
    → darklineTriggered = true ← 触发！
  }

Step 5: 暗线条目加载
  getwi('暗线_总规则')      ← 铁律：AI 绝不主动揭示身份
  
  // 起点涟漪：场景不含'红尘酒家' → 不加载
  // 传说涟漪：暗线触发就加载
  getwi('暗线_主线02_传说涟漪')  ← 碎片演绎规则
  
  // 遗迹涟漪：消息中没有"石碑""金光"等 → 不加载
  // 真相涟漪：不在地府、核心角色不在场 → 不加载
  // 终点涟漪：不在红尘酒家 → 不加载
  
  // 地点碎片：
  currentArea='星辰山脉', currentScene='玄天宗'
  → 匹配条件 → getwi('暗线_碎片_玄天宗')  ← 玄天宗专属碎片

Step 6: AI 生成
  AI 看到：暗线总规则 + 传说涟漪 + 玄天宗碎片
  + 玄天宗世界观 + 角色速览 + 修为元指令
  
  AI 可能让韩青衣在闲聊中说：
  "算卦的时候有时候会看到一些奇怪的东西……上次我推演天机，
  看到了一道金光。不是灵力，不是神通，是更古老的东西。
  我还在琢磨那是什么。"
  
  → 碎片自然融入对话，玩家不知道这是暗线
```

### D2. 暗线如何防止"剧透"——五条铁律的运行时约束

暗线总规则定义了五条铁律，这些铁律作为 `constant: true` 的蓝灯条目，**只要暗线触发就会加载**。AI 在生成时必须遵守：

| 铁律 | 运行时效果 |
|------|----------|
| 绝不主动揭示转世身份 | AI 即使"知道"真相（因为看到了暗线总规则的远古真相），也不会让任何 NPC 说出来 |
| 绝不出现游戏化提示 | AI 不会输出"你发现了一条线索！""暗线推进"等文字 |
| 每角色只知自己那一块 | 韩青衣只能说"我看到了金光"，不能解释金光的含义 |
| 碎片在有理由时自然出现 | 不是触发后立刻出现，而是在对话自然流向该方向时才提到 |
| 洛红尘绝不说破 | 即使玩家追问洛红尘"你等的那个人是谁"，她也只会微笑不语 |

### D3. 动态事件与暗线的互斥机制

EJS 控制器中的互斥用一个简单的 boolean 门控实现：

```javascript
var eventGuideLoaded = false;

// 14 种事件类型依次检测，第一个匹配的加载后设 eventGuideLoaded = true
if (!eventGuideLoaded && (currentEvent === '境界突破' || ...)) {
  getwi('动态事件_境界突破总规则');
  eventGuideLoaded = true;  // ← 后续事件和暗线都被跳过
}
// ... 13 种其他事件 ...

// 暗线被包裹在 if (!eventGuideLoaded) 内部
if (!eventGuideLoaded) {
  // 暗线触发检测和加载
}
```

**为什么互斥？** 因为 token 预算有限。一个动态事件指南（比如境界突破总规则 + 突破阶段细则）可能就有几千字。暗线条目也有几千字。如果同时加载，加上角色档案、宗门设定、地图、记忆、时间线，上下文会爆炸。互斥确保每次对话中要么在"做事"（事件），要么在"听故事"（暗线），不会两者同时出现。

### D4. 好感度在运行时的三层联动

好感度体系不是一个独立模块——它在三个层面同时生效：

**第一层：Zod Schema 的数值约束**

```javascript
好感度: z.coerce.number().transform(v => _.clamp(v, -1000, 1000)).prefault(0)
```

AI 输出的好感度变动经 MVU 处理后，Zod Schema 自动钳制到 [-1000, 1000]。即使 AI 写了 `好感度: 9999`，实际写入变量的也是 1000。

**第二层：好感度参考条目的行为约束**

`角色好感度阶段参考` 是蓝灯常驻条目（constant=true, enabled=true），每次对话都在 AI 上下文中。它告诉 AI：
- 好感度 700 的冷淡角色可能只是"认可你"，不是爱你
- 好感度 800 的温柔角色可能已经心生情愫
- 从红颜知己到情愫暗生必须有特定契机（生死/朝夕相处/特定心动/嫉妒）
- 男性角色终点是"生死之交"，绝不出现爱情线

**第三层：EW Flow 的数值更新**

AI 在主回复中通过 `<UpdateVariable>` 标签输出好感度变化：
```xml
<UpdateVariable>
{
  "女性好感度": {
    "殷冬雪": {
      "好感度": 350,
      "关系阶段": "泛泛之交",
      "重要事件": "在秘境中并肩作战"
    }
  }
}
</UpdateVariable>
```

MVU 解析 → Zod 校验 → 写入 `stat_data` → 状态栏 Vue 应用读取并显示。

### D5. 元指令如何实现"炼气期不能打元婴"

元指令是**按修为境界条件加载的约束规则**。以炼气期为例：

```
能做: 引气入体、五感超常、基础五行术法
不能: 御器飞行、对抗筑基修士
战斗: 碾压凡人，遇筑基必死，遇金丹瞬杀
难度分配: 日常85% 挑战10% 极限5%
遇超阶威胁必须给逃跑机会
```

运行时加载逻辑：
```javascript
if (userCultivation.includes('炼气')) {
  getwi('元指令_修为_炼气期');
} else if (userCultivation.includes('筑基')) {
  getwi('元指令_修为_筑基期');
}
// ... 10 个档位
```

只加载当前境界的元指令，不加载其他档位。这意味着 AI **看不到**更高境界的规则——它只知道炼气期能做什么、不能做什么。如果玩家说"我要飞"，AI 会参考元指令回应"你的修为尚未达到御器飞行的水平"。

突破铁律也写在元指令中：**"修炼进度必须达到100，当前状态必须为'突破在即'"**。这个条件和 Zod Schema 中 `修炼进度: z.coerce.number().transform(v => _.clamp(v, 0, 100))` 联动——进度被钳制在 0-100，必须满 100 才能尝试突破。

### D6. 地图的按大域互斥加载

七个大域地图条目按 `currentDomain` 互斥加载：

```javascript
if (currentDomain === '中央神州' || currentDomain.includes('中央神州')) {
  getwi('地图_中央神州');
} else if (currentDomain === '东荒妖域' || ...) {
  getwi('地图_东荒妖域');
}
// ... else if 链
```

中央神州的地图条目是最长的——18 个宗门、2 个王朝、10+ 特殊势力的完整地理信息。如果不互斥，七个大域的地图全部加载会消耗大量 token。互斥确保 AI 只看到当前所在大域的地图。

### D7. 宗门条目的成对加载

控制器中宗门加载总是成对出现：

```javascript
if (currentArea.includes('天剑峰') || currentScene.includes('天剑圣地')) {
  getwi('天剑圣地');           // 宗门世界观
  getwi('角色速览_天剑圣地');   // 宗门所有角色的速查表
}
```

**为什么要分两个条目？** 因为宗门世界观（地理/核心建筑/宗门特色）是不变的背景信息，而角色速览是一个简短的人名+身份列表，用于让 AI 知道"这个宗门里有谁"。角色的详细档案（调色盘+三面性）则由独立的角色条目存储，只在角色被检测到在场时才加载。

这实现了三层递进的信息加载：
1. **宗门世界观**：当前区域/场景匹配时加载 → 提供地理和背景
2. **角色速览**：同上 → 提供"这里有谁"的信息
3. **角色详细档案**：角色名出现在对话中时加载 → 提供完整人设
